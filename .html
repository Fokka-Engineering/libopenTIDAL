OTDeallocContainer - Deallocate a container from heap

SYNOPSIS

#include <openTIDAL/openTIDAL.h>

void OTDeallocContainer (void *container, enum OTTypes type);

DESCRIPTION

After using a OTContainer returned from a service function one needs to deallocate it. Up to this point values inside the container are still allocated in heap. The container itself is also allocated in heap and needs to be freed.

Specify the type of the container with the OTTypes enum. Available enums: CONTENT_CONTAINER, CONTENT_STREAM_CONTAINER

RETURN VALUE

None

EXAMPLE

    {
        OTSessionClientPair ({clientId}, {clientSecret}); 
        /* do smth with the session if OTSessionClientPair returns 0 */

        OTDeallocContainer (content, CONTENT_CONTAINER);
    }

NAME

OTJsonContainer - OTJson data structure

SYNOPSIS

#include <openTIDAL/openTIDAL.h> 

struct OTJsonContainer
{
    struct OTJsonContainer *next;
    struct OTJsonContainer *prev;
 
    struct OTJsonContainer *child;
    int type;
 
    char *valuestring;
    int valueint;
    char *valueintstring;
    double valuedouble;
 
    char *string;
};

DESCRIPTION

OTJson represents JSON data using the OTJsonContainer struct data type.

An item of this type represents a JSON value. The type is stored in type as a bit-flag (this means that you cannot find out the type by just comparing the value of type).

To check the type of an item, use the corresponding OTJsonIs... function. It does a NULL check followed by a type check and returns a boolean value if the item is of this type.

OTPersistentCreate - Create a persistent configuration file

SYNOPSIS

#include <openTIDAL/openTIDAL.h>

int OTPersistentCreate (const struct OTSessionContainer *const session, const char *const location);

DESCRIPTION

After a successful authentication with the oAuth device authorisation grant (RFC 8628), call this function to create the persistent configuration file. See OTServiceGetDeviceCode and OTServiceGetBearerToken for login polling examples.

OTPersistentCreate allocates a stream of the newly created json session configuration and writes it to the specified file location parameter.

Do not store any additional values inside the configuration file! libopenTIDAL(3) overwrites the configuration file after a successful session oAuth access token renewal.

This function is only available if a file location was specified prior.

RETURN VALUE

If successful OTPersistentCreate returns 0.

If an fopen error (Could not write to file location) or an malloc/parse error occurred, OTPersistentCreate returns -1.

This grant is primarily useful for restricted input devices such as TVs or browser-less applications. Device Authorisation Grant

Call the OTServiceGetDeviceCode service function to request a userCode and a deviceCode. If successful instruct the user to authorise the session with the URI (https://link.tidal.com/ABCDE). While doing that poll the authorisation server with the OTServiceGetBearerToken service function. The polling cycle has a specified minimum interval between each request. The userCode and deviceCode pair has a specified lifetime. If the user was not able to authenticate in this timeframe, request a new code pair. The authorisation server returns the access- and refresh-token and other user values after a successful authentication by the user.

Username/Password and third-party oAuth2.0 authorisation methods are supported.

NAME

OTServiceGetDeviceCode - Request a user- and device-code pair

SYNOPSIS

#include <openTIDAL/openTIDAL.h>

struct OTContentContainer *OTServiceGetDeviceCode (struct OTSessionContainer *session, void *threadHandle);

DESCRIPTION

This service function requests a userCode and a deviceCode from the TIDAL auth server. libopenTIDAL(3) uses the oAuth2.0 device authorisation grant (RFC 8628) to authenticate with the TIDAL API. This grant is primarily useful for restricted input devices such as TVs or browser-less applications. TIDALs oAuth2.0 web-flow (RFC 6749) is reCaptcha v3 secured and thus can't be used outside a web browser. 

Device Authorisation Grant

Call the OTServiceGetDeviceCode service function to request a userCode and a deviceCode. If successful instruct the user to authorise the session with the URI (https://link.tidal.com/ABCDE). #include <openTIDAL/openTIDAL.h>

This service function requests a userCode and a deviceCode from the TIDAL auth server. libopenTIDAL(3) uses the oAuth2.0 device authorisation grant (RFC 8628) to authenticate with the TIDAL API. This grant is primarily useful for restricted input devices such as TVs or browser-less applications. Device Authorisation Grant

Call the OTServiceGetDeviceCode service function to request a userCode and a deviceCode. If successful instruct the user to authorise the session with the URI (https://link.tidal.com/ABCDE). The userCode and deviceCode pair has a specified lifetime. If the user was not able to authenticate in this timeframe, request a new code pair. The authorisation server returns the access- and refresh-token and other user values after a successful authentication by the user.

Username/Password and third-party oAuth2.0 authorisation methods are supported.

NAME

            }

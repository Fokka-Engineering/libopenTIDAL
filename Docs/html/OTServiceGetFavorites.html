NAME

OTServiceGetFavorites - Request user-based metadata

SYNOPSIS

#include <openTIDAL/openTIDAL.h>

struct OTContentContainer *OTServiceGetFavorites (struct OTSessionContainer *session, const char *const suffix, const int limit, const int offset, const char *const order, const char *const orderDirection, void *threadHandle);

DESCRIPTION

The OTServiceGetFavorites service function requests TIDAL metadata. The prefix and suffix parameter are used to specify the endpoint of the request. This service call must have a corresponding call to OTDeallocContainer when the operation is complete. 

Suffixes 
"ids", "albums", "tracks", "videos", "artists", "playlists", "mixes"
 
Pagination

Use the limit and offset parameters to control metadata pagination. Keep in mind that the TIDAL API will error out on certain limits. 

Sorting

order: "DATE", "NAME", "ARTIST", "RELEASE_DATE", "MIX_TYPE"

orderDirection: "ASC", "DESC" 

Thread Handle

You must never share the same handle in multiple threads. You can pass the handles around among threads, but you must never use a single handle from more than one thread at any given time.

Use the session main handle by parsing a NULL pointer.

RETURN VALUE

If no memory allocation error occurred in allocating the OTContentContainer, a pointer to an OTContentContainer will be returned. Otherwise a NULL pointer will be returned.

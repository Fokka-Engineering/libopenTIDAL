NAME

OTServiceGetBearerToken - Request an access- and refresh-token

SYNOPSIS

#include <openTIDAL/openTIDAL.h>

struct OTContentContainer *OTServiceGetBearerToken (struct OTSessionContainer *session, const char *const deviceCode, void *threadHandle);

DESCRIPTION

While the application is waiting for the user to complete the authorization flow OTServiceGetDeviceCode in their browser, the device/application meanwhile begins polling OTServiceGetBearerToken to request an access token.

The application makes a POST request with the deviceCode at the rate specified by interval. The device should continue requesting an access token until the status in the OTContentContainer is SUCCESS. Remember to deallocate the OTContentContainer after each polling-cycle. 

Thread Handle

You must never share the same handle in multiple threads. You can pass the handles around among threads, but you must never use a single handle from more than one thread at any given time.

Use the session main handle by parsing a NULL pointer.

RETURN VALUE

If no memory allocation error occurred in allocating the OTContentContainer, a pointer to an OTContentContainer will be returned. Otherwise a NULL pointer will be returned.

NAME

OTServiceGetPlaylistEntityTag - Request the playlist entity tag

SYNOPSIS

#include <openTIDAL/openTIDAL.h>

char *OTServiceGetPlaylistEntityTag (struct OTSessionContainer *session, const char *const id, void *threadHandle);

DESCRIPTION

The OTServiceGetPlaylistEntityTag service function requests the http header and parses the entity-tag key of the playlist. This entity-tag is used for optimistic concurrency control, as a way to help prevent simultaneous updates of a resource from overwriting each other. Applied with the http header key "If-None-Match". OTServiceGetPlaylistEntityTag() allocates an ASCII String with the http entity-tag value. You need to free the allocated memory after use. Entity-tag handling is integrated into all libopenTIDAL() resource manipulation service functions.

This service call must have a corresponding call to OTDeallocContainer when the operation is complete. 

Thread Handle

You must never share the same handle in multiple threads. You can pass the handles around among threads, but you must never use a single handle from more than one thread at any given time.

Use the session main handle by parsing a NULL pointer.

RETURN VALUE

If no memory allocation error occurred in allocating the ASCII string, a pointer to an ASCII string will be returned. Otherwise a NULL pointer will be returned.
